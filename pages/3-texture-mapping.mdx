# Texture Mapping

Texture mapping in computer graphics is the process of applying a material effect to an object in the scene. The "texture" part is the effect, and the "mapping" part is the mathematical mapping from one space to another. The effect can be any property that we're trying to visualize: color, shininess, bump geometry (bump mapping), or even material presence (to create cut-out surfaces). The most common type of texture maps an image onto a surface to define the color at each point. In practice we implement this in reverse: given a point on the surface, the texture lookup returns the color.

## Constant Color Texture

We start with a `Texture` trait so that any function mapping from a surface point to a color can be used as a texture. The simplest texture is a constant color — but we also add a convenience constructor that takes raw RGB values so callers don't always need to construct a `Color` first.

<Tabs>

    ```rust filename="texture.rs | Texture trait and SolidColor"
    pub trait Texture: Send + Sync {

    }

    pub struct SolidColor {
        albedo: Color,
    }

    impl SolidColor {
        pub fn new(albedo: Color) -> Self {
            Self { albedo }
        }

        pub fn from_rgb(red: f64, green: f64, blue: f64) -> Self {
            Self { albedo: Color::new(red, green, blue) }
        }
    }

    impl Texture for SolidColor {
        fn value(&self, _u: f64, _v: f64, _p: &Point3) -> Color {
            self.albedo

    }
    ```

## Solid Textures: A Checker Texture

A checker texture alternates between two textures based on the sign of a trig function applied to the 3D point coordinates. Because the function depends on the 3D position, it tiles through space — it's a solid (or spatial) texture. The `value()` function ignores `u` and `v`, using only `p`. We also add a convenience constructor that accepts raw colors so callers don't need to wrap them in `SolidColor` manually.

<Tabs>
  <Tab label="Rust">
    ```rust filename="texture.rs | CheckerTexture"
    pub struct CheckerTexture {
        inv_scale: f64,
        even: Arc<dyn Texture>,
        odd: Arc<dyn Texture>,
    }

    impl CheckerTexture {
        pub fn new(scale: f64, even: Arc<dyn Texture>, odd: Arc<dyn Texture>) -> Self {
            Self { inv_scale: 1.0 / scale, even, odd }
        }

        pub fn from_colors(scale: f64, c1: Color, c2: Color) -> Self {
            Self::new(
                scale,
                Arc::new(SolidColor::new(c1)),
                Arc::new(SolidColor::new(c2)),
            )
        }
    }

    impl Texture for CheckerTexture {
        fn value(&self, u: f64, v: f64, p: &Point3) -> Color {
            let x = (self.inv_scale * p.x()).floor() as i32;
            let y = (self.inv_scale * p.y()).floor() as i32;
            let z = (self.inv_scale * p.z()).floor() as i32;
            if (x + y + z) % 2 == 0 {
                self.even.value(u, v, p)
            } else {
                self.odd.value(u, v, p)
            }
        }
    }
    ```
  </Tab>

  <Tab label="C++">
    ```cpp filename="texture.h | checker_texture"
    class checker_texture : public texture {
      public:
        checker_texture(double scale, shared_ptr<texture> even, shared_ptr<texture> odd)
          : inv_scale(1.0 / scale), even(even), odd(odd) {}

        checker_texture(double scale, const color& c1, const color& c2)
          : checker_texture(scale,
                            make_shared<solid_color>(c1),
                            make_shared<solid_color>(c2)) {}

        color value(double u, double v, const point3& p) const override {
            auto xInt = int(std::floor(inv_scale * p.x()));
            auto yInt = int(std::floor(inv_scale * p.y()));
            auto zInt = int(std::floor(inv_scale * p.z()));
            bool isEven = (xInt + yInt + zInt) % 2 == 0;
            return isEven ? even->value(u, v, p) : odd->value(u, v, p);
        }

      private:
        double inv_scale;
        shared_ptr<texture> even, odd;
    };
    ```
  </Tab>
</Tabs>

Now we update `Lambertian` to hold a texture instead of a raw color, calling `tex.value(rec.u, rec.v, rec.p)` at scatter time.

<Tabs>
  <Tab label="Rust">
    ```rust filename="material.rs | Lambertian material with texture"
    pub struct Lambertian {
        tex: Arc<dyn Texture>,
    }

    impl Lambertian {
        pub fn new(albedo: Color) -> Self {
            Self { tex: Arc::new(SolidColor::new(albedo)) }
        }

        pub fn from_texture(tex: Arc<dyn Texture>) -> Self {
            Self { tex }
        }
    }

    impl Material for Lambertian {
        fn scatter(
            &self, r_in: &Ray, rec: &HitRecord,
            attenuation: &mut Color, scattered: &mut Ray,
        ) -> bool {
            let mut scatter_direction = rec.normal + random_unit_vector();
            if scatter_direction.near_zero() {
                scatter_direction = rec.normal;
            }
            *scattered = Ray::new(rec.p, scatter_direction, r_in.time());
            *attenuation = self.tex.value(rec.u, rec.v, &rec.p);
            true
        }
    }
    ```

## Texture Coordinates for Spheres

Constant textures ignore coordinates; spatial textures use 3D position. Now we focus on proper $u,v$ texture coordinates — a 2D parameterization of the surface needed for image textures.

For a sphere we use spherical coordinates. Let $\theta$ be the angle up from the bottom pole (i.e., from $-Y$) and $\phi$ be the angle around the $Y$ axis (from $-X$ toward $+Z$). We normalize to $[0,1]$:

$$u = \frac{\phi}{2\pi}, \quad v = \frac{\theta}{\pi}$$

For a unit sphere centered at the origin these angles are computed as:

$$\phi = \operatorname{atan2}(-z,\, x) + \pi, \quad \theta = \arccos(-y)$$

<Tabs>
  <Tab label="Rust">
    ```rust filename="sphere.rs | get_sphere_uv"
    fn get_sphere_uv(p: &Point3) -> (f64, f64) {
        // p: a given point on the sphere of radius one, centered at the origin.
        // u: angle around the Y axis from X=-1, mapped to [0,1].
        // v: angle from Y=-1 to Y=+1, mapped to [0,1].
        let theta = (-p.y()).acos();
        let phi = (-p.z()).atan2(p.x()) + std::f64::consts::PI;
        let u = phi / (2.0 * std::f64::consts::PI);
        let v = theta / std::f64::consts::PI;
        (u, v)
    }
    ```
  </Tab>

  <Tab label="C++">
    ```cpp filename="sphere.h | get_sphere_uv function"
    static void get_sphere_uv(const point3& p, double& u, double& v) {
        // p: a given point on the sphere of radius one, centered at the origin.
        // u: returned value [0,1] of angle around the Y axis from X=-1.
        // v: returned value [0,1] of angle from Y=-1 to Y=+1.
        auto theta = std::acos(-p.y());
        auto phi = std::atan2(-p.z(), p.x()) + pi;
        u = phi / (2*pi);
        v = theta / pi;
    }
    ```
  </Tab>
</Tabs>

We call `get_sphere_uv` inside the sphere's `hit()` method and store the results in `rec.u` and `rec.v`.

<Tabs>
  <Tab label="Rust">
    ```rust filename="sphere.rs | Sphere UV coordinates from hit"
    fn hit(&self, r: &Ray, ray_t: Interval, rec: &mut HitRecord) -> bool {
        // ... (intersection test) ...
        rec.t = root;
        rec.p = r.at(rec.t);
        let outward_normal = (rec.p - self.center.at(r.time())) / self.radius;
        rec.set_face_normal(r, outward_normal);
        let (u, v) = get_sphere_uv(&outward_normal);
        rec.u = u;
        rec.v = v;
        rec.mat = self.mat.clone();
        true
    }
    ```

## Accessing Texture Image Data

For image textures we load an image file and sample it at $u,v$ coordinates. We clamp out-of-range values and flip $v$ since image row 0 is at the top but $v=0$ is at the bottom. A solid cyan return guards against missing image files.

<Tabs>
  <Tab label="Rust">
    ```rust filename="texture.rs | ImageTexture"
    pub struct ImageTexture {
        image: RgbImage,
    }

    impl ImageTexture {
        pub fn new(filename: &str) -> Self {
            let image = image::open(filename)
                .expect("Could not open image file")
                .into_rgb8();
            Self { image }
        }
    }

    impl Texture for ImageTexture {
        fn value(&self, u: f64, v: f64, _p: &Point3) -> Color {
            if self.image.height() == 0 {
                return Color::new(0.0, 1.0, 1.0);
            }
            let u = u.clamp(0.0, 1.0);
            let v = 1.0 - v.clamp(0.0, 1.0);
            let i = (u * self.image.width() as f64) as u32;
            let j = (v * self.image.height() as f64) as u32;
            let i = i.min(self.image.width() - 1);
            let j = j.min(self.image.height() - 1);
            let pixel = self.image.get_pixel(i, j);
            let scale = 1.0 / 255.0;
            Color::new(
                scale * pixel[0] as f64,
                scale * pixel[1] as f64,
                scale * pixel[2] as f64,
            )
        }
    }
    ```
  </Tab>

  <Tab label="C++">
    ```cpp filename="texture.h | image_texture"
    class image_texture : public texture {
      public:
        image_texture(const char* filename) : image(filename) {}

        color value(double u, double v, const point3& p) const override {
            if (image.height() <= 0) return color(0,1,1);
            u = interval(0,1).clamp(u);
            v = 1.0 - interval(0,1).clamp(v);
            auto i = int(u * image.width());
            auto j = int(v * image.height());
            auto pixel = image.pixel_data(i, j);
            auto color_scale = 1.0 / 255.0;
            return color(color_scale*pixel[0], color_scale*pixel[1], color_scale*pixel[2]);
        }

      private:
        rtw_image image;
    };
    ```
  </Tab>
</Tabs>
