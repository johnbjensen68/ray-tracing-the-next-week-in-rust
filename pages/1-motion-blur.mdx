# Motion Blur

## Introduction of SpaceTime Ray Tracing

When you decided to ray trace, you decided that visual quality was worth more than run-time. Motion blur is a great example of this philosophy. In a real camera, the shutter remains open for a short time interval, during which the camera and objects in the world may move. To accurately reproduce such a camera shot, we seek an average of what the camera senses while its shutter is open.

We can get a random estimate of this by sending each ray at a random time while the shutter is open. As long as objects are where they are supposed to be at that time, the ray can determine the color for that sample. This is the essence of how we will implement motion blur.

## Managing Time

We need to track time in our ray type. We'll add a field to store the time at which the ray was cast. The camera will generate rays at random times within the shutter interval.

<Tabs>
  <Tab label="Rust">
    ```rust filename="ray.rs | Adding time to Ray"
    pub struct Ray {
        pub origin: Point3,
        pub direction: Vec3,
        pub time: f64,
    }

    impl Ray {
        pub fn new(origin: Point3, direction: Vec3, time: f64) -> Ray {
            Ray { origin, direction, time }
        }

        pub fn at(&self, t: f64) -> Point3 {
            self.origin + t * self.direction
        }
    }
    ```
  </Tab>

  <Tab label="C++">
    ```cpp filename="ray.h | Adding time to ray"
    class ray {
      public:
        ray() {}

        ray(const point3& origin, const vec3& direction, double time)
          : orig(origin), dir(direction), tm(time) {}

        const point3& origin() const  { return orig; }
        const vec3& direction() const { return dir; }
        double time() const           { return tm; }

        point3 at(double t) const {
            return orig + t*dir;
        }

      private:
        point3 orig;
        vec3 dir;
        double tm;
    };
    ```

## Updating the Camera to Simulate Motion Blur

The camera needs to generate rays at random times within its shutter interval. We add `shutter_open` and `shutter_close` times to the camera, and when generating a ray we pick a random time in that range.

<Tabs>
  <Tab label="Rust">
    ```rust filename="camera.rs | Camera with shutter interval"
    pub struct Camera {
        // ... existing fields ...
        pub shutter_open: f64,
        pub shutter_close: f64,
    }

    impl Camera {
        pub fn get_ray(&self, s: f64, t: f64) -> Ray {
            let rd = self.lens_radius * Vec3::random_in_unit_disk();
            let offset = self.u * rd.x() + self.v * rd.y();
            let time = self.shutter_open
                + random_double() * (self.shutter_close - self.shutter_open);
            Ray::new(
                self.origin + offset,
                self.lower_left_corner + s * self.horizontal + t * self.vertical
                    - self.origin
                    - offset,
                time,
            )
        }
    }
    ```
  </Tab>

  <Tab label="C++">
    ```cpp filename="camera.h | Camera with shutter interval"
    class camera {
      public:
        // ... constructor with open/close times ...
        ray get_ray(double s, double t) const {
            vec3 rd = lens_radius * random_in_unit_disk();
            vec3 offset = u * rd.x() + v * rd.y();
            return ray(
                origin + offset,
                lower_left_corner + s*horizontal + t*vertical - origin - offset,
                random_double(shutter_open, shutter_close)
            );
        }
      private:
        double shutter_open, shutter_close;
    };
    ```
  </Tab>
</Tabs>

## Adding Moving Spheres

We create a `MovingSphere` that linearly interpolates its center position between `center0` at `time0` and `center1` at `time1`. At a given ray time, we compute where the sphere's center is and perform the usual intersection math.

<Tabs>
  <Tab label="Rust">
    ```rust filename="moving_sphere.rs | MovingSphere"
    pub struct MovingSphere {
        pub center0: Point3,
        pub center1: Point3,
        pub time0: f64,
        pub time1: f64,
        pub radius: f64,
        pub material: Arc<dyn Material>,
    }

    impl MovingSphere {
        pub fn center(&self, time: f64) -> Point3 {
            self.center0
                + ((time - self.time0) / (self.time1 - self.time0))
                    * (self.center1 - self.center0)
        }
    }
    ```
  </Tab>

  <Tab label="C++">
    ```cpp filename="moving_sphere.h | moving_sphere class"
    class moving_sphere : public hittable {
      public:
        moving_sphere(
            point3 cen0, point3 cen1, double time0, double time1,
            double r, shared_ptr<material> m)
          : center0(cen0), center1(cen1), time0(time0), time1(time1),
            radius(r), mat(m) {}

        point3 center(double time) const {
            return center0 + ((time - time0) / (time1 - time0)) * (center1 - center0);
        }
      private:
        point3 center0, center1;
        double time0, time1;
        double radius;
        shared_ptr<material> mat;
    };
    ```
  </Tab>
</Tabs>

## Tracking the Time of Ray Intersection

The `hit` function for a moving sphere uses `ray.time()` to compute the sphere's center at the moment of intersection. The rest of the math stays the same as for a stationary sphere.

<Tabs>
  <Tab label="Rust">
    ```rust filename="moving_sphere.rs | hit implementation"
    impl Hittable for MovingSphere {
        fn hit(&self, r: &Ray, ray_t: Interval, rec: &mut HitRecord) -> bool {
            let center = self.center(r.time);
            let oc = r.origin - center;
            let a = r.direction.length_squared();
            let half_b = oc.dot(r.direction);
            let c = oc.length_squared() - self.radius * self.radius;
            let discriminant = half_b * half_b - a * c;
            if discriminant < 0.0 {
                return false;
            }
            // ... rest of hit logic ...
            true
        }
    }
    ```
  </Tab>

  <Tab label="C++">
    ```cpp filename="moving_sphere.h | hit function"
    bool moving_sphere::hit(
        const ray& r, interval ray_t, hit_record& rec) const
    {
        point3 center = this->center(r.time());
        vec3 oc = r.origin() - center;
        auto a = r.direction().length_squared();
        auto half_b = dot(oc, r.direction());
        auto c = oc.length_squared() - radius*radius;
        auto discriminant = half_b*half_b - a*c;

    }
    ```

## Putting Everything Together

With moving spheres and a camera that generates rays with random times, we can render a scene with motion blur. The resulting image captures the sense of movement: each moving sphere appears smeared in the direction of its motion.

The key insight is that we don't change our integration loop at all — we just evaluate color at a random time for each sample, and averaging handles the rest.
